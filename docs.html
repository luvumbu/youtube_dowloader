1 Installer XAMPP

XAMPP fournit Apache (serveur web) et PHP dont l'application a besoin.

Telechargement

  1. Aller sur apachefriends.org
  2. Telecharger la version Windows (PHP 8.x)
  3. Lancer l'installateur xampp-windows-x64-8.x.x-installer.exe

Installation

  1. Chemin d'installation : laisser C:\xampp (par defaut)
  2. Composants : cocher au minimum Apache et PHP (MySQL optionnel)
  3. Terminer l'installation et lancer XAMPP Control Panel
  4. Cliquer sur Start a cote de Apache

Ouvrir http://localhost/ dans le navigateur. Si la page XAMPP s'affiche, c'est bon.

Si le port 80 est deja utilise (Skype, IIS, etc.), XAMPP ne pourra pas demarrer Apache. Fermer l'application qui utilise le port ou changer le port Apache dans C:\xampp\apache\conf\httpd.conf.

2 Installer Python

Python est necessaire pour faire fonctionner yt-dlp.

Telechargement

  1. Aller sur python.org/downloads
  2. Telecharger la derniere version de Python 3.x pour Windows
  3. Lancer l'installateur

Installation

  1. IMPORTANT : cocher la case "Add Python to PATH" en bas de l'ecran d'installation
  2. Cliquer sur "Install Now"
  3. Attendre la fin et fermer

Ouvrir un terminal (Win+R → cmd) et taper :

python --version

Doit afficher Python 3.x.x. Si "non reconnu", redemarrer le PC et reessayer.

3 Installer yt-dlp

yt-dlp est l'outil qui telecharge les videos/audios depuis YouTube.

Installation via pip

Ouvrir un terminal (cmd) et executer :

pip install yt-dlp

Trouver le chemin de yt-dlp

Apres l'installation, il faut connaitre le chemin exact de l'executable :

where yt-dlp

La commande affiche quelque chose comme :

C:\Users\TON_USER\AppData\Local\Programs\Python\Python3xx\Scripts\yt-dlp.exe

Note bien ce chemin ! Tu en auras besoin a l'etape 6 pour configurer l'application.

Tester que ca marche :

yt-dlp --version

4 Installer FFmpeg

FFmpeg convertit les formats audio/video (MP3, AAC, etc.).

Methode 1 : via winget (recommandee)

Ouvrir un terminal en administrateur et executer :

winget install Gyan.FFmpeg

Methode 2 : installation manuelle

  1. Aller sur gyan.dev/ffmpeg/builds
  2. Telecharger ffmpeg-release-full.7z
  3. Extraire le dossier (ex: dans C:\ffmpeg\)
  4. Le dossier bin\ contient ffmpeg.exe, ffprobe.exe, ffplay.exe

Trouver le chemin du dossier bin

where ffmpeg

Si installe via winget, le chemin ressemble a :

C:\Users\TON_USER\AppData\Local\Microsoft\WinGet\Packages\Gyan.FFmpeg_...\ffmpeg-x.x-full_build\bin\

Si installe manuellement :

C:\ffmpeg\bin\

Note bien le chemin du dossier bin (pas le chemin de ffmpeg.exe, le dossier qui le contient). Tu en auras besoin a l'etape 6.

Verifier l'installation :

ffmpeg -version

Si ffmpeg n'est pas reconnu dans le terminal, il faut l'ajouter au PATH Windows :
Parametres → Systeme → A propos → Parametres avances → Variables d'environnement → Path → Ajouter le chemin du dossier bin.

5 Copier les fichiers du projet

Placer l'application dans le bon dossier de XAMPP.

Copier le projet

  1. Copier tout le dossier youtube du PC source
  2. Coller dans C:\xampp\htdocs\ sur le nouveau PC
  3. Le resultat doit etre C:\xampp\htdocs\youtube\

Verifier les dossiers necessaires

Ces dossiers doivent exister. S'ils manquent, les creer :

mkdir C:\xampp\htdocs\youtube\downloads
mkdir C:\xampp\htdocs\youtube\data

Fichiers de donnees

Si les fichiers JSON n'existent pas, les creer vides :

echo {"folders":[],"items":[]} > C:\xampp\htdocs\youtube\data\library.json
echo {"profiles":[]} > C:\xampp\htdocs\youtube\data\profiles.json

Ne PAS copier les fichiers du dossier downloads\ du PC source (sauf si tu veux garder les fichiers telecharges). Les fichiers temporaires (.log, .done) ne doivent pas etre copies.

6 Configurer les chemins

Adapter la configuration aux chemins du nouveau PC. C'est l'etape la plus importante.

Ouvrir le fichier de configuration

Ouvrir C:\xampp\htdocs\youtube\classes\Config.php dans un editeur de texte (Notepad, VS Code, etc.).

Modifier les 3 chemins

Remplacer les chemins en haut du fichier par ceux de ton PC :

Ouvrir classes\Config.php et modifier uniquement ces 3 lignes (en haut du fichier) :

// Ligne 13 — Remplacer par le resultat de : where yt-dlp
const YTDLP_PATH = 'C:\\Users\\TON_USER\\...\\Scripts\\yt-dlp.exe';

// Ligne 16 — Remplacer par le resultat de : where ffmpeg
// ATTENTION : c'est le DOSSIER bin, pas ffmpeg.exe
const FFMPEG_PATH = 'C:\\Users\\TON_USER\\...\\ffmpeg-x.x-full_build\\bin';

// Ligne 19 — Ne change pas si XAMPP installe dans C:\xampp
const PHP_PATH = 'C:\\xampp\\php\\php.exe';

Le reste du fichier Config.php n'a pas besoin d'etre modifie. Seuls ces 3 chemins changent d'un PC a l'autre.

Attention aux antislash ! En PHP, il faut doubler les \ dans les chemins Windows :
C:\Users\... devient C:\\Users\\...

Verifie bien que chaque \ est double dans les chemins.

Exemple concret

Si ton nom d'utilisateur Windows est jean et que tu as installe Python 3.12 :

const YTDLP_PATH = 'C:\\Users\\jean\\AppData\\Local\\Programs\\Python\\Python312\\Scripts\\yt-dlp.exe';
const FFMPEG_PATH = 'C:\\ffmpeg\\bin';
const PHP_PATH = 'C:\\xampp\\php\\php.exe';

7 Verification

Tester que tout fonctionne correctement.

1. Verifier Apache

  1. Ouvrir XAMPP Control Panel
  2. Cliquer Start a cote de Apache (le fond devient vert)
  3. Ouvrir http://localhost/ → la page XAMPP doit s'afficher

2. Verifier l'application

  1. Ouvrir http://localhost/youtube/
  2. L'interface du YouTube Downloader doit apparaitre

3. Tester un telechargement

  1. Coller une URL YouTube dans le champ de saisie
  2. Cliquer sur Telecharger
  3. Les infos de la video doivent s'afficher (titre, miniature)
  4. La barre de progression doit avancer
  5. Le fichier doit apparaitre dans la bibliotheque une fois termine

4. Verifier les chemins en cas de probleme

Ouvrir un terminal et tester chaque outil :

:: Tester PHP
C:\xampp\php\php.exe -v

:: Tester yt-dlp (remplacer par ton chemin)
"C:\Users\TON_USER\...\yt-dlp.exe" --version

:: Tester ffmpeg (remplacer par ton chemin)
"C:\Users\TON_USER\...\bin\ffmpeg.exe" -version

Si les 3 commandes affichent une version, la configuration est correcte.

8 Demarrage automatique (optionnel)

Lancer XAMPP automatiquement au demarrage de Windows pour ne plus avoir a y penser.

Creer le raccourci de demarrage

  1. Ouvrir l'Explorateur de fichiers
  2. Aller dans C:\xampp\
  3. Clic droit sur xampp_start.exe → Creer un raccourci
  4. Renommer le raccourci en XAMPP AutoStart
  5. Appuyer sur Win+R, taper shell:startup et valider
  6. Deplacer le raccourci dans le dossier qui s'ouvre

Au prochain demarrage de Windows, XAMPP (Apache) se lancera automatiquement en arriere-plan. L'application sera accessible sur http://localhost/youtube/ sans rien faire.

Raccourci bureau (optionnel)

Pour creer un raccourci sur le bureau qui ouvre directement l'application :

  1. Clic droit sur le Bureau → Nouveau → Raccourci
  2. Emplacement : http://localhost/youtube/
  3. Nom : YouTube Downloader

! Depannage

Solutions aux problemes les plus courants.

La page affiche une erreur PHP

Cause : Les chemins dans Config.php sont incorrects.

Solution : Verifier chaque chemin avec where yt-dlp et where ffmpeg dans le terminal. S'assurer que les \ sont doubles.

La video est reconnue mais le telechargement ne demarre pas

Cause : Le worker PHP ne se lance pas en arriere-plan.

Solution : Verifier que le chemin PHP_PATH dans Config.php est correct. Tester dans le terminal :

C:\xampp\php\php.exe -v

yt-dlp retourne une erreur

Cause : yt-dlp n'est pas a jour (YouTube change souvent ses protections).

Solution : Mettre a jour yt-dlp :

pip install -U yt-dlp

Le fichier telecharge n'a pas de son (video)

Cause : FFmpeg n'est pas trouve pour fusionner audio+video.

Solution : Verifier que FFMPEG_PATH dans Config.php pointe bien vers le dossier bin\ contenant ffmpeg.exe.

Apache ne demarre pas (port 80 occupe)

Cause : Un autre programme utilise le port 80.

Solution : Dans le terminal administrateur, trouver le programme :

netstat -ano | findstr :80

Puis fermer le programme en question, ou changer le port Apache.

Erreur "Video protegee" ou telechargement qui echoue

Cause : Certaines videos sont protegees par YouTube (DRM, restrictions regionales, etc.).

Solution : L'application retente automatiquement jusqu'a 2 fois. Si ca echoue quand meme, la video n'est probablement pas telechargeable. Essayer de mettre a jour yt-dlp.
